# Cómo contribuir

¡Gracias por tu interés! Este proyecto es software libre y toda ayuda es bienvenida.

## Reportar bugs

Abre un [issue](../../issues) e incluye:

- Qué esperabas que pasara y qué pasó.
- Pasos para reproducirlo.
- Tu sistema operativo y versión del proyecto.
- Logs o capturas si aplica.

## Proponer cambios

1. Haz un *fork* del repo y crea una rama descriptiva (`fix/lectura-usb`, `feat/exportar-json`).
2. Haz tus cambios en commits pequeños y con mensajes claros.
3. Si tocas lógica, **agrega o actualiza tests**.
4. Corre los tests localmente antes de abrir el PR.
5. Abre el Pull Request explicando **qué** cambia y **por qué**.

## Estilo

- Sigue el estilo del código existente; no reformatees archivos enteros de más.
- Mantén la documentación en **español e inglés** cuando exista en ambos.
- Un PR = un objetivo. Cambios sin relación, en PRs separados.

## Antes de proponer una feature grande

Abre primero un issue para discutirla — así evitamos que inviertas tiempo en algo que no encaje con la dirección del proyecto.

---

*Al contribuir aceptas que tu aporte se publique bajo la misma licencia del proyecto.*
